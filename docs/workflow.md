# Workflow: создание футбольной Data Platform

Документ описывает последовательность работ, чеклисты и критерии готовности для построения платформы данных по архитектуре из `data_platform_architecture.md`.

## Цель и конечный результат
- **Цель**: развернуть минимально жизнеспособную платформу (HDFS + Hive Metastore + Spark + Trino + Airflow) с конвейером от raw до gold и базовыми витринами.
- **Готовность (MVP)**: работают ingestion по 1–2 источникам, ETL в `silver`, 1–2 витрины в `gold`, доступ из Trino/BI, мониторинг и бэкапы базового уровня.

## Общие принципы
- Идемпотентность, воспроизводимость, конфигурация через `.env` и `docker-compose`.
- Директории и таблицы в `snake_case`; партиции — `key=value`.
- Данные: слои `raw` → `silver` → `gold`; хранение Parquet (Snappy/ZSTD).
